## Pass 1 ŌĆö NOT_READY

Three blocking findings, all upheld on the merits:

- **B1 ŌĆö the `profiles` column had no vocabulary to validate against.**
  `kit.json` has no `profiles` field; the vocabulary would be defined by
  ADR-060, a *draft*. So the manifest would ship a column whose valid values
  were undefined the day it landed ŌĆö either unvalidated free text (drift from
  day one) or a dependency inversion against the ADR's own sequencing.
- **B2 ŌĆö the ownership taxonomy was asserted before its "owner" ADR.**
  `kit-owned | generated | user-seeded` was described as what ADR-059's planner
  *will* read. If ADR-059's review revised it, every manifest row would be wrong
  on arrival.
- **B3 ŌĆö the parser contract was unspecified, and the cited precedent does not
  parse.** `criteria.md` is a table scripts *cite* but none parses
  (`check-plan-eval.sh` reimplements the criteria; `check-plan-criteria-drift`
  guards by mtime). A permissive parser that skipped malformed rows would
  recreate the warn-and-continue failure this ADR exists to end ŌĆö one layer up.

Non-blocking: `schemaVersion` semantics in a hand-edited file (N1),
`--allow-missing` governance (N3), the overclaimed "single source of truth"
(N4), the unenforced `since-version` column (N5), the overstated "three sources"
framing (N6), and a query on whether `check-install-render` was invented scope
(N7).

## Resolution

The fixes **specify** Option B; they do not change it. The Decision (a markdown
manifest + thin parser) is unchanged.

- **B1/B2 ŌĆö this ADR now owns both vocabularies.** The ownership set is closed
  here and ADR-059 *adopts* it; changing it requires a superseding ADR, not a
  silent redefinition. The profile set ships as exactly `{ full }`, enforced as
  a **closed** vocabulary, and ADR-060 extends it on acceptance. This makes the
  dependency run one way, which is the only way the "sequence 061 first"
  argument holds.
- **B3 ŌĆö a Parser contract section was added:** strict parsing (a malformed row
  is a hard error naming the row, never skipped); **no fallback** (an
  unparseable manifest aborts the install rather than reverting to a hardcoded
  list, which would quietly restore Option A); required ŌćÆ fail-fast with a clean
  target, optional ŌćÆ warn-and-continue; and `--allow-missing` as an explicit,
  loud, test-only flag.
- N1/N4/N5/N6 were adopted into the text (versioning as a real parser contract;
  the source-of-truth claim scoped to installation and ownership; `since-version`
  marked explicitly unvalidated; the "three sources" framing corrected to one
  structured declaration surrounded by uncontrolled copies).
- **N7 was refuted with evidence, not accepted:** `bin/check-install-render`
  exists (18,834 bytes, dated Jun 21) and predates this ADR. It is an existing
  consumer, not smuggled scope. The evidence was handed back to the reviewer in
  pass 2.

## Durable lesson

**An ADR may not depend on a vocabulary defined by a downstream draft.** If ADR
X sequences before Y *because* Y consumes X, then every enumerated value X
exposes must be closed by X. Otherwise the sequencing is decorative: X ships a
column only Y can validate, and the "single source of truth" is a forward
reference to an unaccepted document. The cure is cheap when caught at review ŌĆö
name the closed set, say who may extend it ŌĆö and expensive afterwards, because
the fix lands as a data migration.

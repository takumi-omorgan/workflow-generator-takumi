# 2026-07-13 — Adversarial review of ADR-057 (public export integrity gate)

## What was reviewed

`design/adr/adr-057-public-export-integrity-gate.md`, for acceptance under
issue #45 / PR #46.

**Reviewer model (observed, not assumed):** `qwen/qwen3.7-plus`, served via
OpenRouter through the `hermes` CLI. Availability was probed before the run
(auth + network OK, model self-identified as `qwen/qwen3.7-plus`); no fallback
model was used at any point.

This took **two passes**, and the first pass is the important part of the
record.

## Pass 1 — READY, but on a false premise

The first ADR-057 draft motivated the gate with three asserted defects in the
published repo (`olivermorgan2/claude-workflow-kit`): (1) an identity mismatch
making the published install instructions install a *different* repo, (2)
internal exhaust (`archive/phase-1/`, `notes/`, the `CLAUDE.md` contributor
section) shipped in the public tree, and (3) no post-publish enforcement.

Qwen returned **READY, 4/5, zero blockers** — but explicitly noted it had no
repo access ("I have no direct observation, but the claims are internally
consistent"), and its **#1 finding demanded the premise be verified**, warning
that if the export pipeline was never the problem then "the defect is upstream
of ADR-057's fix entirely."

Acting on that finding is what caught the error. A full audit of the published
repo via the GitHub API, at its then-current HEAD (commit `1c0eba3`, dated
2026-06-29 — the same tree the original external review looked at, and older
than the 2026-07-08 ADR draft, so this is *not* "true then, fixed since"):

| Claim | Verdict | Evidence |
|---|---|---|
| 1. Identity mismatch | **FALSE** | `README.md`, `docs/install.md`, and `bin/bootstrap-workflow-kit` all reference `olivermorgan2/claude-workflow-kit` — the repo they ship in. Zero `takumi-omorgan/*` strings in the published tree. |
| 2. Internal exhaust shipped | **FALSE** | Recursive audit of every published path: no `archive/`, no `notes/`, no root `design/`, no `.hermes/`. `prompts/` holds only `_template.md`, per contract. Published `CLAUDE.md` has 0 occurrences of the contributor-notes section (source has 1) — correctly stripped. |
| 3. No post-publish enforcement | **TRUE, but latent** | There is genuinely no `bin/verify-published`. But the claimed consequence — silent drift — has not occurred: the published file set is exactly the 258 files `bin/export-public` produces today, zero extra, zero missing. |

**The ADR-056 export contract is being honoured by the published artifact.**
Two of three motivating defects did not exist. A `READY` verdict conditional on
an unverified premise is not a licence to merge — the gate did its job precisely
by refusing to stop at the verdict.

## Premise correction

The ADR was **rewritten from scratch** around the one real gap rather than
patched. Per the Hermes rule against preserving false claims in accepted ADR
text, the falsified claims appear **nowhere** in the accepted ADR — they are
recorded here, in this receipt, as review history. Issue #45's Context (which
carried the same three claims) is corrected by a comment on the issue.

The rewrite also **dropped the identity-rewrite half of the original Option B**
(a `--public-repo=OWNER/NAME` export flag). That work was commissioned by
defect 1, which does not exist; ADR-056's TRANSFORM rules plus verifier check E
already produce and enforce correct public identity strings, as the published
tree confirms. It is now explicitly out of scope in the ADR.

## Pass 2 — the rewrite (the accepted text)

Re-reviewed with a self-contained pack: the rewritten ADR, ADR-056,
`docs/publishing.md`, issue #45, PR #46 state, the audit evidence above, and
live validation output. The reviewer was told the first draft's premises had
been falsified and instructed to check for residual false claims and to reject
the ADR outright if the shrunken motivation no longer justified the decision.

**Verdict: READY. Score: 4/5. Blockers: 0.**

Four findings, all `minor`, **all applied** to the ADR before acceptance:

1. **Date ambiguity** — the audit-date/commit-date parenthetical could be
   misread as dating the commit to 2026-07-13. Reworded to separate the audit
   date (2026-07-13) from the commit date (2026-06-29).
2. **Unstated derivation** — the "published tag resolves to default-branch
   HEAD" assertion follows from the *current* publish model in
   `docs/publishing.md`, not from any invariant ADR-056 states. Added a note
   saying so, and that the assertion must be revised if the publish model ever
   gains release branches.
3. **Overstated consequence** — "the manual audit becomes a repeatable command"
   was too broad. Narrowed to the mechanical part (tree equality, excluded
   paths, identity strings, version pin); holistic human judgment is explicitly
   not automated.
4. **Threat model** — digest comparison catches accidental divergence, not
   adversarial tampering. Made explicit that accepting this ADR does not close
   the supply-chain-integrity question; asset signing stays deferred to ADR-055.

Reviewer's rationale, verbatim in substance: the revision "successfully excises
the two false premises"; every remaining factual claim is backed by the audit
evidence; the gap is real (the verified local tree and the live remote tree are
separately derived and nothing asserts equivalence); it fits ADR-054's
script-the-spine decision rule; it contradicts no accepted ADR. *"The shrunken
premise does not weaken the ADR — it clarifies it."*

## Why the final ADR is acceptable

The decision no longer rests on any claim about a broken published repo. It
rests on a structural fact that is directly observable in the accepted ADR-056
and `docs/publishing.md`: `bin/export-public` **never pushes** (ADR-056 Decision
1), so publishing is a manual human sequence, and the tree that is *verified*
(`dist/`, local) and the tree that is *live* on the remote are two separately
derived artifacts with nothing asserting they are the same. Every leg of the
export contract is machine-checked except the last one.

That the contract has held so far is established only by a one-off human audit —
which is exactly the kind of work ADR-054's decision rule ("if two competent runs
must produce byte-identical output, it is a deterministic script") says should
not be left to vigilance. ADR-057 is accepted as **defence-in-depth on a latent
gap**, not as a fix for an active defect, and its text says so plainly.

## Durable lessons

- **A `READY` verdict from a reviewer without repo access is conditional on its
  own premise findings.** Qwen scored the first draft 4/5 while simultaneously
  asking whether the premise was true. Treating the score as the output and the
  finding as noise would have enshrined two false claims in an immutable
  accepted ADR and commissioned real implementation work against a defect that
  does not exist. **Read the findings, not just the verdict.**
- **This is the second time this exact failure mode has been recorded.** The
  2026-06-23 Codex review produced 3 of 7 false positives from a stale read, and
  its durable lesson was *"verify findings against the tree at its current HEAD,
  not a remembered version."* An external review of the public repo then repeated
  it, and ADR-057's first draft inherited the result. **When a review asserts an
  observable defect, observe it before designing against it** — the cost of the
  check is minutes; the cost of skipping it is an immutable wrong decision.
- **Give adversarial reviewers the evidence, and permission to reject.** Pass 2's
  pack included the falsification table and explicitly authorised the verdict
  "the premise shrank, so drop the ADR." A reviewer that cannot reject cannot
  ratify.
